De-Novo

Accessible Communication Platform

Empowering People with Disabilities

Full-Stack Application • Django + React

Why De-Novo?

The Challenge

Deaf, blind, and mute users lack integrated platforms designed specifically for their accessibility needs.

Our Solution

An all-in-one platform combining real-time chat, voice AI, mood support, and enterprise-grade security.

Core Features

  • Real-Time Chat - WebSocket-powered messaging with E2E encryption
  • Voice AI Suite - Text-to-Speech, Speech-to-Text, Sentiment Analysis
  • Mood Zone - Ambient sounds, mood tracking, recommendations
  • Security - 2FA, Peeping Tom detection, privacy shields

Real-Time Chat System

Backend (Django)

Django Channels + Redis for WebSocket messaging. MongoDB for flexible message storage.

Frontend (React)

Real-time message rendering, sentiment indicators, read receipts, E2E encryption badge.

Google Cloud AI Services

  • Text-to-Speech (TTS) - Multiple voices, adjustable rate/pitch
  • Speech-to-Text (STT) - Continuous listening, punctuation auto-insert
  • Sentiment Analysis - Emotion detection in real-time chat

Accessibility Features (Visual)

  • Font size scaling (14px - 32px)
  • High contrast mode toggle
  • Color blindness filters (4 types)
  • Dark/Light theme support
  • Reduced motion options

Accessibility Features (Audio)

  • Text-to-Speech with voice selection
  • Speech rate and pitch control
  • Continuous speech recognition
  • Language selection support
  • ARIA labels on all interactive elements

Data Storage Strategy

MySQL

User profiles, accessibility settings, contacts, security logs

MongoDB

Messages, mood history, sentiment data (flexible schema)

Security Architecture

  • End-to-End Encryption for messages
  • Two-Factor Authentication (2FA)
  • Peeping Tom Detection (webcam intrusion alerts)
  • JWT-based authentication
  • CORS protection

Mood Zone: Emotional Wellness

Ambient soundscapes: Nature, Music, White Noise

Visual sound wave animations with mood tracking

AI-powered recommendations based on mood patterns

Relaxation session timers and mood journal

Onboarding Experience

  • Step 1: Basic info (name, email, password)
  • Step 2: Disability type selection
  • Step 3: Accessibility preferences
  • Step 4: Profile setup with voice guidance

Frontend Technologies

  • React 18+ with React Router v6 navigation
  • Tailwind CSS for responsive styling
  • Web Speech API for voice interactions
  • Context API for state management
  • Custom hooks for accessibility features

Backend Technologies

  • Django 4.2+ with REST Framework
  • Django Channels for WebSocket support
  • Redis for caching and message queuing
  • Celery for async task processing
  • Google Cloud AI APIs integration

API Architecture

  • /api/users/ - Authentication, profile management
  • /api/chat/ - Message CRUD, conversation management
  • /api/accessibility/ - User preference management
  • /api/ai/ - TTS, STT, sentiment analysis

Architectural Reasoning

Dual Database: MySQL for structured data, MongoDB for flexible message schema

WebSockets: Real-time messaging without polling overhead

Cloud AI: Outsource ML complexity, focus on UX

Contact Management Features

Alphabetical contact lists with voice search

Filter contacts by disability type

Block/unblock functionality

QR code sharing for easy connections

User Dashboard

  • Quick action tiles (Chat, Mood Zone, Settings)
  • Recent conversations preview
  • Online friends list
  • Notification center
  • Floating accessibility toolbar

Scalability Strategy

Redis Caching: User preferences and session data

Celery Tasks: Background processing for AI services

Database Indexing: Optimize queries on frequently accessed fields

Quality Assurance
